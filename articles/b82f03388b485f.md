## はじめに

普段、Homebrewの更新する際に次のコマンドを実行しています。

```bash
brew update && brew upgrade && brew cleanup
```

これでも問題はないのですが、少々打つのが面倒です。

この面倒を解消するために、今回はTaskfileに追加してコマンドを簡略化を行います。

ちなみに参考にした記事もあります。
（むしろこの記事の内容を実践した方が良い場合もあります。）

https://zenn.dev/takaha4k/articles/brew-cleanup-command

## 環境

この記事を書いている時点のTaskfileのバージョンは以下の通りです。

```bash
task --version
3.51.1
```

## タスクの定義

今回実行したいコマンドを次の通りです。

- `brew update`
- `brew upgrade`
- `brew upgrade --cask`
- `brew cleanup`
- `brew doctor`

上記コマンドを定義したTaskfileはこんな感じです。

```yaml
# yaml-language-server: $schema=https://taskfile.dev/schema.json

version: '3'

tasks:
  brewup:
    desc: Updating Homebrew
    cmds:
      - echo "==> Running health check (before)..."
      - brew doctor
      - echo "==> Updating Homebrew..."
      - brew update
      - echo "==> Upgrading formula..."
      - brew upgrade || true
      - echo "==> Upgrading cask..."
      - brew upgrade --cask || true
      - echo "==> Cleaning up..."
      - brew cleanup
      - echo "==> Running health check (after)..."
      - brew doctor
      - echo "✅ Done"
    silent: true
```

ポイントは`brew doctor`を最初と最後に実行することで壊れていないかをチェックするところです。
チェックで何かあれば止まるようにしています。
逆に`brew upgrade`と`brew upgrade --cask`ではアプリの都合（アプリを閉じていない、権限など）で失敗する可能性もあるので、`|| true`を付けて処理を止めないようにしています。

## タスクの実行

Taskfile.ymlを配置したディレクトリであれば以下で実行はできます。

```bash
task brewup
```

ただ、実行場所が決まってくると使い勝手が悪いので、ホームディレクトリにTaskfile.ymlを作成し、実行時には必ず`--global`(もしくは`-g`)オプションを付けて実行します。

```bash
task -g brewup
```

こうすることでどこからでも実行することが可能になります。

## まとめ

今回は定期実行するHomebrewのコマンドをTaskfileに定義してみました。
